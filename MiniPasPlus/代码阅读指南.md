## 先记住一句话

这个项目做的事情很简单：

1. 先把源码拆成一个个小块（Token）
2. 再检查这些小块能不能组成合法程序
3. 同时生成中间代码（四元式）
4. 再把中间代码做一点优化
5. 最后整理成界面可以展示的数据

## 第 0 步：先看入口

文件：`src/main.cpp`

你只要看懂三件事：

1. `source` 是要编译的 MiniPas 程序文本
2. `CompilerFacade compiler;` 是总控对象
3. `compiler.compileAndRun(source)` 会把整条流程跑完

如果失败，打印 `errorMessage`。
如果成功，打印统计数量。

## 第 1 步：看总控（最重要）

文件：`src/CompilerFacade.cpp`

函数：`CompileResult CompilerFacade::compileAndRun(...)`

把这个函数当“总指挥”：

1. 调 Lexer
2. 调 Parser
3. 调优化器
4. 调展示数据生成器
5. 把结果打包到 `CompileResult`

你讲的时候可以说：

“我不在 main 里直接写词法和语法，而是统一交给 CompilerFacade 来调度。”

## 第 2 步：看 Lexer（字符怎么变 Token）

文件：`src/Lexer.cpp`

重点函数：

1. `tokenize()`：总扫描循环
2. `readIdentifierOrKeyword()`：识别关键字/标识符
3. `readNumber()`：识别常量
4. `readOperatorOrDelimiter()`：识别运算符和界符

你要记住的判断顺序：

1. 先跳过空白和注释
2. 看当前字符：字母 / 数字 / 其他符号
3. 分别走不同读取函数
4. 最后补一个 EOF Token

## 第 3 步：看 Parser（递归下降）

文件：`src/Parser.cpp`

先看 `parse()`，它是 Parser 总入口。

再按这个顺序看：

1. `parseProgram`
2. `parseTypeDeclPart`
3. `parseFunctionDecl`
4. `parseVarDeclPart`
5. `parseStmt`（语句分发）
6. `parseAssignStmt / parseIfStmt / parseWhileStmt`
7. `parseExpression / parseTerm / parseFactor`
8. `emit`
9. `buildBasicBlocks`

### 三个核心点

1. 语法规则是“一个规则一个函数”
2. 语义检查在解析过程中顺便做（类型、声明、参数）
3. 每次 `emit` 就是在生成一条四元式

## 第 4 步：看优化器

文件：`src/BackendOptimization.cpp`

函数：`optimizeByBasicBlocks`

它做的是“局部优化示例”：

1. 在每个基本块里做替换
2. 做代数化简
3. 做公共子表达式复用

注意：这是局部优化示例，不是完整工业级全局优化器。

## 第 5 步：看展示数据模块

文件：

- `src/TargetCodeDemo.cpp`
- `src/ActivationRecordDemo.cpp`
- `src/RuntimeResultDemo.cpp`

这些模块不改变编译结果，只是把结果转成“方便 GUI 展示的表格行”。

## 第 6 步：看 CompileResult（结果总盒子）

文件：`include/CompileResult.h`

它是所有阶段的汇总结构。

你要重点认识这些字段：

1. `tokens`
2. `identifierTable / constantEntries`
3. `symbols / functionTable / parameterTable`
4. `quadruples / optimizedQuadruples / basicBlocks`
5. `parserTrace / parserSteps / parserActions`

## 最后：上台讲解模板（可直接背）

“这个项目入口在 main。main 只负责调用 CompilerFacade。CompilerFacade 是总控，会依次调用 Lexer、Parser、优化器和展示数据模块。Lexer 把源码拆成 Token。Parser 用递归下降检查语法语义并生成四元式。优化器按基本块做局部优化。最后所有结果放进 CompileResult 统一输出。这样主流程清晰，便于对照阅读。”
