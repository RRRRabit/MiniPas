# MiniPas VM 可用指令表（运行层规范）

最后更新：2026-05-16

目标：保留现有展示层；新增独立运行层（VM）时，按本表执行。  
原则：展示文本怎么写都行，VM 只认结构化指令。

## 1. 统一指令格式（运行层）

每条指令统一为四元结构：

`[OP, A, B, C]`

- `OP`：操作码（如 `LD`、`ADD`、`FJ`）
- `A/B/C`：操作数字段（不用时填 `_`）

约定的操作数类型：

- `R0/R1/...`：寄存器
- `#n`：立即数（如 `#5`、`#3.14`）
- `@name`：符号内存槽（变量名、临时名、`RETVAL` 等）
- `Lx`：标签（跳转目标）

## 2. 可用指令总表（语义已定）

| 类别 | 指令 | 执行语义（通俗中文） |
|---|---|---|
| 取值 | `LD A, B, _` | 把 `B` 的值取出来放到 `A`。常见是 `A` 为寄存器。 |
| 存值 | `ST A, B, _` | 把 `A` 的值写到 `B`。常见是写回变量槽。 |
| 赋值 | `MOV A, B, _` | `A = B`，语义上等于简单复制。 |
| 算术 | `ADD A, B, C` | `A = B + C` |
| 算术 | `SUB A, B, C` | `A = B - C` |
| 算术 | `MUL A, B, C` | `A = B * C` |
| 算术 | `DIV A, B, C` | `A = B / C` |
| 关系 | `LT A, B, C` | `A = (B < C)`，结果是真/假 |
| 关系 | `GT A, B, C` | `A = (B > C)` |
| 关系 | `EQ A, B, C` | `A = (B == C)` |
| 关系 | `LE A, B, C` | `A = (B <= C)` |
| 关系 | `GE A, B, C` | `A = (B >= C)` |
| 关系 | `NE A, B, C` | `A = (B != C)` |
| 逻辑 | `AND A, B, C` | `A = (B && C)` |
| 逻辑 | `OR A, B, C` | `A = (B \|\| C)` |
| 逻辑 | `NO A, B, _` | `A = !B` |
| 条件跳转 | `FJ A, Lx, _` | 如果 `A` 为假，跳转到 `Lx` |
| 条件跳转 | `TJ A, Lx, _` | 如果 `A` 为真，跳转到 `Lx` |
| 无条件跳转 | `JMP Lx, _, _` | 直接跳到 `Lx` |
| 传参 | `ARG A, _, _` | 把 `A` 压入参数区（按顺序） |
| 调用 | `CALL A, n, _` | 调用函数 `A`，`n` 为实参数量 |
| 返回 | `RET A, _, _` | 函数返回，把 `A` 作为返回值 |
| 取返回值 | `MOVRET A, _, _` | `A = RETVAL` |
| 停机 | `HALT _, _, _` | 程序结束，VM 停止执行 |

## 3. 当前项目最小落地子集（建议先实现）

先把下面这组做完，就能跑当前主要流程（表达式/分支/循环/函数调用）：

`LD ST MOV ADD SUB MUL DIV LT GT EQ LE GE NE FJ JMP ARG CALL RET MOVRET HALT`

下面这些可后续再做：

`TJ AND OR NO`

## 4. 与展示层的关系（重要）

- 展示层：你现在 UI 里显示的“目标代码字符串”，继续保留。
- 运行层：新增 `Instruction` 数组给 VM 执行，不依赖展示字符串。
- 渲染方式：展示层可以把 `Instruction` 再格式化回“课程风格文本”。

也就是说：

- VM 跑的是结构化指令。
- 界面看的是格式化文本。
- 两层解耦，互不拖累。

## 5. 错误与边界处理（运行层必须）

- `DIV`：除数为 0 要抛运行时错误。
- 跳转：目标标签不存在要报错。
- `CALL`：函数入口不存在要报错。
- `RET`：在主程序帧执行 `RET` 视为程序结束或报错（二选一，需固定约定）。

## 6. 一条指令的示例（便于实现）

```text
[ADD, R0, @x, #1]
```

含义：`R0 = x + 1`

```text
[FJ, R0, L_end, _]
```

含义：如果 `R0` 为假，跳到 `L_end`
