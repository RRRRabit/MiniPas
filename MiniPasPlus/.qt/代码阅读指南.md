# MiniPas 代码阅读指南（单人讲解版）

这份文档是给“刚学完 C++、但要答辩一个编译器项目”的你看的。目标不是让你成为这个项目的维护者，而是让你能在答辩时讲清楚：代码从哪里开始、每一步干什么、功能到底在哪些函数里实现。


> 注释版更新说明：现在所有核心代码和 GUI 展示代码都补了中文注释，所以行数比之前更多。阅读时优先按函数名找位置，行号只作为大概定位。

先记住一句总纲：

```text
源程序字符串
  -> Lexer 词法分析，得到 Token
  -> Parser 语法/语义分析，得到符号表和四元式
  -> BackendOptimization 优化四元式
  -> VM 模块执行生成的指令（答辩时当黑盒，不读源码）
  -> 输出最终变量值
```


> 目录调整说明：`VirtualMachine.h/.cpp` 已经从 `include/` 和 `src/` 移到 `MiniPasPlus/VM/`。你现在答辩阅读时不读 VM 源码，只把它当作“执行 VM 指令并输出变量结果”的黑盒模块。

## 3. 第二步：读 `CompilerFacade.cpp`

路径：`MiniPasPlus/src/CompilerFacade.cpp`

这是最重要的总控文件。你答辩开场可以直接从它讲。

### 行 1-10：包含哪些模块

你会看到它包含：

#include "CompilerBackendStages.h"

这说明总控会用到：

- `Parser`：语法/语义分析。
- `BackendOptimization`：四元式优化。
- `VirtualMachine`：VM 执行器，黑盒调用，不读源码。
- `Interpreter`：VM 失败时的回退执行器，也不作为重点阅读。

### 行 13-70：错误处理辅助函数

这里主要有：

```cpp
setCompileError(...)
tryParseLineColumn(...)
```

这两段不是核心编译逻辑，只是把错误信息整理成人能看的形式。可以粗读。

### 行 72：核心函数 `compileAndRun`

这是总控主线，一定要认真读。

```cpp
CompileResult CompilerFacade::compileAndRun(const std::string& sourceCode)
```

你可以按下面 6 段读。

### 行 76-87：词法分析

核心代码意思是：

```cpp
Lexer lexer(sourceCode);
std::vector<Token> tokens = lexer.tokenize();
```

它把源程序字符串交给 Lexer，得到 Token 序列，并把关键字表、界符表、标识符表、常量表放进 `finalResult`。

答辩说法：

> 第一阶段是词法分析，把字符流变成 Token 序列，同时建立标识符表和常量表。

### 行 89-110：词法错误检查

这里遍历 Token，如果发现 `TokenType::ERROR`，就说明词法阶段已经失败。

答辩说法：

> 词法分析不会直接崩溃，而是把非法字符做成 ERROR Token，总控发现后统一返回错误信息。

### 行 113-123：语法语义分析

核心代码：

```cpp
Parser parser(tokens);
CompileResult parseResult = parser.parse();
```

Parser 会生成：

- 符号表 `symbols`
- record 类型表 `recordTypes`
- array 类型表 `arrayTypes`
- 函数表 `functionTable`
- 参数表 `parameterTable`
- 四元式 `quadruples`
- 基本块 `basicBlocks`

答辩说法：

> 第二阶段 Parser 检查 Token 是否符合语法，同时做语义检查，并生成四元式。

### 行 125-127：四元式优化

核心代码：

```cpp
finalResult.optimizedQuadruples =
    backend::optimizeByBasicBlocks(finalResult.quadruples, finalResult.basicBlocks);
```

答辩说法：

> 优化阶段以基本块为单位，对四元式做局部优化，例如常量折叠和公共子表达式消除。

### 后半段：交给 VM 模块运行（不读 VM 源码）

核心代码：

```cpp
finalResult.vmInstructions =
    vm::generateVmInstructions(finalResult.quadruples,
                               finalResult.functionTable,
                               finalResult.parameterTable);
```

注意：这里用的是原始四元式，不是优化后四元式。这样控制流和函数调用更稳定，便于答辩解释。

答辩说法：

> 这里开始进入运行模块，我只需要知道它把前面生成的四元式相关结果交给 VM，VM 最终返回变量值；VM 源码不作为阅读重点。

### VM 执行和回退（只理解作用）

核心意思：

- 总控会调用 VM 模块执行。
- 如果 VM 抛异常，就用 `Interpreter` 回退执行。
- 最终变量值放入 `runtimeValues`。

答辩说法：

> 最后一阶段是运行时执行。这里我不展开 VM 内部，只说明它接收目标指令并输出最终变量值；如果 VM 执行出错，就回退到简单解释器。

## 5. 第四步：读 `Token.h`

路径：`MiniPasPlus/include/Token.h`

这个文件定义词法分析的结果。

### `enum class TokenType`

Token 类型包括：

- `KEYWORD`：关键字，比如 `program`、`var`
- `IDENTIFIER`：标识符，比如变量名 `a`
- `CONSTANT`：常量，比如 `123`、`3.14`
- `DELIMITER`：界符，比如 `;`、`,`、`(`、`)`
- `OPERATOR`：运算符，比如 `+`、`:=`
- `END_OF_FILE`：文件结束
- `ERROR`：非法字符或非法词

### `struct Token`

一个 Token 记录：

- 类型
- 字面值 `lexeme`
- 行号
- 列号
- 编码信息

答辩说法：

> Token 是词法分析输出的最小单元。后面的 Parser 不直接看字符，而是读取 Token。

## 6. 第五步：读 `Lexer.cpp`

路径：`MiniPasPlus/src/Lexer.cpp`

这是词法分析实现。

### 行 6-23：构造函数

这里初始化：

你只要知道关键字和界符是在这里登记的。

### 行 25：核心函数 `Lexer::tokenize`

这是词法分析主函数。建议重点读行 25-162。

它的结构是：

```cpp
while (!isAtEnd()) {
    skipWhitespace();
    if (skipComment()) continue;
    ...
}
```

它从左到右扫描源代码。

### 行 45-63：识别关键字和标识符

如果当前字符是字母，就不断读取字母或数字。

读完一个单词后：

- 如果在关键字表里，就是 `KEYWORD`。
- 否则就是 `IDENTIFIER`，加入标识符表。

例子：

```pascal
var a: integer;
```

会识别出：

```text
var -> KEYWORD
a -> IDENTIFIER
integer -> KEYWORD
```

### 行 65-103：识别数字常量

如果当前字符是数字，就读取整数或实数。

支持：

- `123`
- `3.14`

遇到 `array[1..5]` 时，要注意 `..` 是数组上下界符号，不能把 `1.` 当成实数。

### 行 105-125：识别字符常量

处理：

```pascal
'a'
```

如果单引号不完整，会生成错误 Token。

### 行 127-154：识别符号

处理：

- `:=`
- `!=`
- `..`
- `+`
- `-`
- `*`
- `/`
- `;`
- `,`
- `(`
- `)`

### 行 215-247：跳过空白和注释

`skipWhitespace()` 跳过空格、换行、制表符。

`skipComment()` 支持两种注释：

```pascal
{ block comment }
// line comment
```

### 行 251-276：维护标识符表和常量表

`addIdentifier()`：如果标识符已经出现过，返回原编号；否则加入表。

`addConstant()`：常量同理。

## 8. 第七步：读 `Parser.cpp`

路径：`MiniPasPlus/src/Parser.cpp`

这是项目里最值得读的核心文件。不要一次读完，按下面区间读。

### 行 19-49：`Parser::parse`

这是 Parser 总入口。

它做的事：

1. 清空旧状态。
2. 调用 `parseProgram()`。
3. 检查文件是否结束。
4. 调用 `buildBasicBlocks()`。
5. 把符号表、类型表、函数表、四元式等结果放进 `result_`。

答辩说法：

> Parser 的入口是 `parse()`，它负责初始化分析状态，然后从文法开始符号 `PROGRAM` 开始递归下降分析。

### 行 51-64：`parseProgram`

文法：

```text
PROGRAM -> program id DECL_PART COMPOUND_STMT .
```

代码顺序也完全对应：

1. 读 `program`。
2. 读程序名。
3. 符号表加入程序名。
4. 生成 `program` 四元式。
5. 分析声明部分。
6. 分析复合语句。
7. 读结尾 `.`。
8. 生成 `end` 四元式。

这是最适合展示“递归下降”的函数。

### 行 66-111：类型声明入口

包括：

- `parseDeclPart()`
- `parseTypeDeclPart()`
- `parseTypeDeclList()`
- `parseTypeDecl()`

重点理解：

```pascal
type
  arr = array[1..5] of integer;
```

Parser 会把 `arr` 作为类型名存进类型表。

### 行 113-176：record 和 array 类型

重点函数：

- `parseTypeSpec()`
- `parseRecordTypeDecl()`
- `parseArrayTypeDecl()`
- `parseFieldDeclList()`
- `parseFieldDecl()`

你要能讲：

- record 类型保存字段列表。
- array 类型保存下界、上界、元素类型。
- 数组上下界如果下界大于上界，会报错。

答辩说法：

> 自定义类型不直接生成运行结果，而是记录在 `TypeTable` 里，后面声明变量和检查字段/下标时会用。

### 行 178-188：基本类型

`parseBasicType()` 支持：

- `integer`
- `real`
- `char`
- `boolean`

### 行 190-240：函数声明

重点函数：

```cpp
parseFunctionDecl()
```

它做的事：

1. 读 `function`。
2. 读函数名。
3. 读形参列表。
4. 读返回类型。
5. 把函数加入符号表。
6. 分析函数局部变量。
7. 分析函数体。
8. 生成 `ret` 四元式。
9. 把函数信息加入函数表。

特别注意：

```pascal
function add(a,b: integer): integer;
begin
  add := a + b
end;
```

Pascal 风格函数返回值是“给函数名赋值”。

答辩说法：

> 函数声明阶段会记录函数入口四元式位置、参数个数、返回类型等信息，后续函数调用检查和运行阶段需要用。

### 行 242-270：函数参数

包括：

- `parseParamList()`
- `parseParam()`

重点：

- 参数会加入参数表。
- 参数也会加入符号表。
- 当前主要支持值参调用。

### 行 272-344：变量声明

包括：

- `parseVarDeclPart()`
- `parseVarDeclList()`
- `parseVarDecl()`
- `parseIdList()`

重点：

```pascal
var
  a,b: integer;
```

`parseIdList()` 先读 `a,b`，然后 `parseTypeName()` 读类型。

每个变量会加入符号表：

- 普通变量：`variable`
- record 变量：`record variable`
- array 变量：`array variable`
- 函数内局部变量：`local ... of 函数名`

### 行 346-375：类型名解析

`parseTypeName()` 支持：

- 基本类型：`integer`、`real`、`char`、`boolean`
- 自定义类型：record 类型名、array 类型名

如果使用未声明类型，会报错。

### 行 377-412：复合语句和语句列表

`parseCompoundStmt()` 对应：

```pascal
begin
  ...
end
```

`parseStmtList()` 负责读多条语句，语句之间用 `;` 分隔。

### 行 414-441：`parseStmt`

这是语句分发器。

它根据当前 Token 判断下一条语句是什么：

- `begin`：复合语句。
- `if`：条件语句。
- `while`：循环语句。
- `id(`：函数调用语句。
- 其他标识符开头：赋值语句。

答辩说法：

> `parseStmt` 就像语句路由器，根据当前 Token 决定调用哪个具体语句分析函数。

### 行 443-454：赋值语句

`parseAssignStmt()` 对应：

```pascal
a := expression
```

流程：

1. 解析左值。
2. 读取 `:=`。
3. 解析右侧表达式。
4. 检查类型是否能赋值。
5. 生成 `:=` 四元式。

核心四元式：

```text
(:=, 表达式结果, _, 左值)
```

### 行 462-474：while

`parseWhileStmt()` 对应：

```pascal
while condition do statement
```

生成四元式：

```text
(wh, _, _, _)
(关系运算, 左, 右, t1)
(do, t1, _, _)
循环体四元式
(we, _, _, _)
```

你要能讲：

> `wh/do/we` 是循环结构标记，后面运行阶段会根据这些标记完成跳转执行。

### 行 476-491：if

`parseIfStmt()` 对应：

```pascal
if condition then statement else statement
```

生成四元式：

```text
(关系运算, 左, 右, t1)
(if, t1, _, _)
then 分支
(el, _, _, _)
else 分支
(ie, _, _, _)
```

没有 else 时，就没有 `el` 分支。

### 行 493-509：条件表达式

`parseCondition()` 支持：

- `<`
- `>`
- `=`
- `!=`

流程：

1. 解析左表达式。
2. 读取关系运算符。
3. 解析右表达式。
4. 创建 boolean 临时变量。
5. 生成关系运算四元式。

例如：

```pascal
a < b
```

生成：

```text
(<, a, b, t1)
```

### 行 511-540：左值

`parseLeftValue()` 支持：

- 普通变量：`a`
- record 字段：`p.age`
- 数组元素：`a[i]`

暂不支持：

```pascal
a[i].field
```

答辩说法：

> 左值分析不仅要读语法，还要检查变量是否声明、字段是否合法、数组下标类型是否正确。

### 行 542-576：表达式和项

`parseExpression()` 处理：

```text
+ -
```

`parseTerm()` 处理：

```text
* /
```

这样自然实现了乘除优先级高于加减。

例如：

```pascal
a + b * 4
```

会先在 `parseTerm()` 里处理 `b * 4`，再回到 `parseExpression()` 处理 `a + t1`。

### 行 578-636：因子

`parseFactor()` 支持：

- 标识符
- 函数调用
- 数组元素
- record 字段
- 常量
- 括号表达式

这里是表达式最底层。

### 行 638-697：函数调用

`parseFunctionCall()` 做：

1. 检查函数是否声明。
2. 读取实参列表。
3. 找到对应形参列表。
4. 检查实参数量。
5. 检查实参类型。
6. 为每个实参生成 `param` 四元式。
7. 生成 `call` 四元式。

例如：

```pascal
z := add(4, 5)
```

会生成：

```text
(param, 4, vf, _)
(param, 5, vf, _)
(call, add, 2, t1)
(:=, t1, _, z)
```

## 9. 第八步：读 `ParserSupport.cpp`

路径：`MiniPasPlus/src/ParserSupport.cpp`

这个文件是 Parser 的工具箱。很多“看起来像魔法”的操作都在这里。

### 行 8-30：读 Token 的基础函数

- `peek()`：看当前 Token。
- `previous()`：看上一个 Token。
- `isAtEnd()`：是否到文件末尾。
- `check()`：当前 Token 是否符合预期。

你可以把 `check` 理解成“只看一眼，不往前走”。

### 行 32-40：`match`

`match()` 的特点：

- 如果匹配，就 `current_++` 前进一步。
- 如果不匹配，不报错，返回 false。

适合可选结构，比如：

```cpp
if (match(TokenType::KEYWORD, "else")) { ... }
```

因为 `else` 可以有，也可以没有。

### 行 42-56：`consume`

`consume()` 的特点：

- 必须匹配。
- 不匹配就报错。

适合必须出现的符号，比如：

```cpp
consume(TokenType::KEYWORD, "program", "程序必须以 program 开头");
```

答辩说法：

> `match` 用于可选符号，`consume` 用于必须出现的符号。这是递归下降常见写法。

### 行 58-67：错误报告

`errorAtToken()` 会抛出异常，信息包含行号、列号和当前 Token。

### 行 69-80：基本类型和语句开头判断

- `isBasicTypeToken()`：判断是否为 `integer/real/char/boolean`。
- `isStatementStart()`：判断当前 Token 能不能作为语句开头。

### 行 83-110：变量和 record 字段检查

- `checkVariableDeclared()`：变量是否声明。
- `checkLeftValue()`：左值是否合法。

如果写：

```pascal
p.xxx := 1
```

但 `xxx` 不是 record 字段，就会在这里报错。

### 行 112-130：数组元素类型检查

`resolveArrayElementType()` 做三件事：

1. 数组变量必须声明。
2. 下标表达式必须是 `integer`。
3. 基础变量必须真的是数组类型。

然后返回数组元素类型。

### 行 132-179：左值类型解析

`resolveLValueType()` 根据左值字符串判断类型。

支持：

- `a`
- `p.age`
- `arr[i]`

它会查符号表和类型表，返回真实类型。

### 行 181-195：算术类型合并

`mergeNumericType()` 用于表达式：

```pascal
a + b
```

规则：

- 两边必须是 `integer` 或 `real`。
- 只要有一边是 `real`，结果就是 `real`。
- 两边都是 `integer`，结果就是 `integer`。

### 行 197-208：赋值兼容

`canAssign()` 判断：

- 类型完全相同，可以赋值。
- `integer` 可以赋给 `real`。
- 其他情况不行。

例如：

```pascal
realVar := intVar
```

允许。

```pascal
intVar := realVar
```

不允许。

### 行 210-233：临时变量

`newTemp()` 生成：

```text
t1, t2, t3...
```

表达式计算中间结果会放到临时变量。

例如：

```pascal
c := a + b * 4
```

中间的 `b * 4` 就会放进 `t1`。

### 行 235-238：生成四元式

`emit()` 是四元式生成唯一入口。

```cpp
result_.quadruples.push_back({op, arg1, arg2, result});
```

你答辩时可以重点展示这个函数，因为它非常短，但代表中间代码生成。

### 行 240-415：基本块划分

`buildBasicBlocks()` 用于优化。

基本块就是一段连续四元式，只有一个入口和一个出口。

它会把这些位置作为基本块入口：

- 第 0 条四元式。
- if/else 的分支入口。
- while 循环体入口。
- 循环结束后的下一条。

答辩说法：

> 优化通常以基本块为单位，因为基本块内部没有复杂跳转，做局部优化更简单。

## 16. 老师常问问题和标准回答

### 问：为什么要先做词法分析？

答：

> 源代码本质是字符流，直接按字符做语法分析很复杂。词法分析先把字符归类成 Token，比如关键字、标识符、常量、运算符。Parser 后面只需要处理 Token 序列。

### 问：递归下降体现在哪里？

答：

> 体现在 `Parser.cpp` 里。每个文法规则都有对应函数，例如 `parseProgram`、`parseStmt`、`parseExpression`。函数内部按照文法顺序调用其他 parse 函数。

### 问：表达式优先级怎么处理？

答：

> 用函数层级处理。`parseExpression` 处理加减，`parseTerm` 处理乘除，`parseFactor` 处理变量、常量和括号。因为 `parseExpression` 会先调用 `parseTerm`，所以乘除优先级自然高于加减。

### 问：语义检查在哪里？

答：

> 语义检查主要在 Parser 阶段完成。比如 `checkVariableDeclared` 检查变量声明，`resolveArrayElementType` 检查数组下标，`mergeNumericType` 检查算术类型，`canAssign` 检查赋值兼容。

### 问：四元式怎么生成？

答：

> Parser 在分析语句和表达式时调用 `emit` 生成四元式。表达式中间结果用 `newTemp` 生成临时变量保存。

### 问：if 和 while 怎么实现？

答：

> 我重点讲编译阶段：Parser 先生成结构化四元式标记，比如 `if/el/ie`、`wh/do/we`。这些标记表达了分支和循环的控制关系。运行阶段交给独立 VM 模块处理，我不展开 VM 源码。

### 问：函数调用怎么实现？

答：

> 我重点讲编译阶段：Parser 检查函数是否声明、实参数量和类型是否匹配，然后生成 `param`、`call`、`ret` 四元式。运行阶段交给独立 VM 模块，我不读 VM 内部实现。

### 问：数组和 record 是真正内存寻址吗？

答：

> 目前类型检查已经实现，数组和 record 信息保存在类型表里；运行层采用符号名近似表示，还不是完整地址化内存模型。

### 问：优化做到什么程度？

答：

> 当前是基本块内局部优化，包括常量折叠、复制传播、公共子表达式消除、代数化简和删除无用临时变量。没有做跨基本块全局优化。
