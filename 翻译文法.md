## 1. 程序结构

原文法：

`<PROGRAM> → program id <DECL_PART> <COMPOUND_STMT> .`

翻译文法：

`<PROGRAM> → program id a1 <DECL_PART> <COMPOUND_STMT> . a2`

语义动作：

- `a1`：记录程序名，填符号表主表（kind=program），生成程序开始四元式 `(program, pname, _, _)`
- `a2`：生成程序结束四元式 `(end, pname, _, _)`

实现对应：

- `parseProgram()`

## 2. 变量声明

原文法：

- `<VAR_DECL> → <ID_LIST> : <TYPE_NAME> ;`
- `<ID_LIST> → id { , id }`

翻译文法：

- `<VAR_DECL> → <ID_LIST> a3 : <TYPE_NAME> a4 ; a5`

语义动作：

- `a3`：把本次声明的标识符名序列暂存（待统一写表）
- `a4`：确定当前声明类型 `T`，取长度 `len(T)`
- `a5`：对 `<ID_LIST>` 中每个名字：
  - 重定义检查
  - 填写符号表（name/type/kind/address/size）
  - 更新偏移地址（全局 `nextAddress_` 或函数局部 `currentLocalOffset_`）

实现对应：

- `parseVarDecl()`
- `parseIdList()`
- `typeTable_.getTypeSize(...)`

## 3. 赋值语句

原文法：

`<ASSIGN_STMT> → <LEFT_VALUE> := <EXPRESSION>`

翻译文法：

`<ASSIGN_STMT> → <LEFT_VALUE> := <EXPRESSION> a6`

语义动作：

- `a6`：
  - 计算左值类型 `LT` 与表达式类型 `RT`，做赋值兼容检查
  - 生成赋值四元式：`(:=, <EXPRESSION>.place, _, <LEFT_VALUE>.place)`

实现对应：

- `parseAssignStmt()`
- `resolveLValueType(...)`
- `canAssign(...)`
- `emit(":=", ...)`

## 4. 算术表达式

原文法（等价形）：

- `<EXPR> → <EXPR> + <TERM> | <EXPR> - <TERM> | <TERM>`
- `<TERM> → <TERM> * <FACTOR> | <TERM> / <FACTOR> | <FACTOR>`
- `<FACTOR> → id | const | ( <EXPR> )`

翻译文法（动作示意）：

- `<EXPR> → <EXPR> + <TERM> a7 | <EXPR> - <TERM> a8 | <TERM>`
- `<TERM> → <TERM> * <FACTOR> a9 | <TERM> / <FACTOR> a10 | <FACTOR>`

语义动作：

- `a7/a8/a9/a10`（统一模式）：
  - 新建临时变量 `t = newTemp(typeMerge(left.type, right.type))`
  - 生成四元式：`(op, left.place, right.place, t)`
  - 返回 `{place=t, type=mergedType}`

实现对应：

- `parseExpression()`
- `parseTerm()`
- `parseFactor()`
- `newTemp(...)`
- `mergeNumericType(...)`
- `emit(op, ...)`

## 5. 条件与控制流

### 5.1 条件表达式

原文法：

`<CONDITION> → <EXPRESSION> relop <EXPRESSION>`

翻译文法：

`<CONDITION> → <EXPRESSION> relop <EXPRESSION> a11`

语义动作：

- `a11`：
  - 新建布尔临时变量 `tb`
  - 生成关系四元式：`(relop, left.place, right.place, tb)`
  - 返回 `tb` 作为条件结果

实现对应：

- `parseCondition()`

### 5.2 if 语句

原文法：

`<IF_STMT> → if <CONDITION> then <STMT> [ else <STMT> ]`

翻译文法（与现实现一致的标记法）：

`<IF_STMT> → if <CONDITION> a12 then <STMT> [ else a13 <STMT> ] a14`

语义动作：

- `a12`：生成 `(if, cond.place, _, _)`
- `a13`：生成 `(el, _, _, _)`
- `a14`：生成 `(ie, _, _, _)`

实现对应：

- `parseIfStmt()`

### 5.3 while 语句

原文法：

`<WHILE_STMT> → while <CONDITION> do <STMT>`

翻译文法：

`<WHILE_STMT> → while a15 <CONDITION> a16 do <STMT> a17`

语义动作：

- `a15`：生成 `(wh, _, _, _)`
- `a16`：生成 `(do, cond.place, _, _)`
- `a17`：生成 `(we, _, _, _)`

实现对应：

- `parseWhileStmt()`

## 6. 函数调用与返回

原文法（简化）：

- `<CALL_EXPR> → id ( <ARG_LIST> )`
- `<ARG_LIST> → <EXP> { , <EXP> } | ε`
- `<FUNCTION_DECL> → ... <COMPOUND_STMT> ;`

翻译文法：

- `<CALL_EXPR> → id ( <ARG_LIST> ) a18`
- `<ARG_LIST> → <EXP> a19 { , <EXP> a19 } | ε`
- `<FUNCTION_DECL> → ... <COMPOUND_STMT> a20 ;`

语义动作：

- `a19`：每个实参生成 `param` 四元式：`(param, arg.place, vf, _)`
- `a18`：
  - 形参与实参数量/类型检查
  - 如有返回值先分配临时变量 `tr`
  - 生成调用四元式：`(call, fname, argc, tr)`
- `a20`：
  - 函数体末尾生成返回四元式：`(ret, fname, _, _)`

实现对应：

- `parseFunctionCall(...)`
- `parseFunctionDecl()`

## 7. 左值语义（数组/记录）

原文法（当前实现）：

- `<LEFT_VALUE> → id | id[index] | id.field`

翻译动作要点：

- `id[index]`：检查下标类型必须为 `integer`，并校验基对象为数组类型
- `id.field`：校验基对象为 record，且字段存在
- 通过 `resolveLValueType(...)` 返回左值类型用于赋值检查

实现对应：

- `parseLeftValue()`
- `resolveArrayElementType(...)`
- `checkLeftValue(...)`
- `resolveLValueType(...)`
