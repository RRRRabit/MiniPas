# 已实现的功能（MiniPas）

最后更新：2026-05-16

本文档只记录“代码中已经实现并可用”的功能，按模块划分。  
后续每次功能改动，都应同步更新这份文档。

## 1. 工程与界面

- 已实现 Qt 桌面应用入口与主窗口。
- 已实现源代码编辑区、示例代码下拉选择、打开源文件按钮、编译运行按钮、清空按钮。
- 已实现多标签结果展示：
- 词法分析
- 符号表系统
- 原始四元式
- 中间代码及优化
- 目标代码
- VM运行结果
- 已实现状态栏提示（成功/失败）与编译错误弹窗。
- 已实现多处点击联动高亮（Token、标识符、关键字、界符、常数、符号表单元格联动源码高亮）。

## 2. 词法分析（Lexer）

- 已实现 Token 分类：
- `KEYWORD`、`IDENTIFIER`、`CONSTANT`、`DELIMITER`、`OPERATOR`、`END_OF_FILE`、`ERROR`
- 已实现关键字识别：`program var integer real char begin end type record array of function boolean while do if then else`
- 已实现界符/运算符识别：`, : ; := * / + - . ( ) = [ ] .. < > !=`
- 已实现标识符表（I 表）与常数表（C 表）构建。
- 已实现常数类型识别：
- 整数常量
- 实数常量
- 字符常量（如 `'a'`）
- 已实现注释跳过：
- `{ ... }`
- `// ...`
- 已实现非法词法项检测并返回 `ERROR` Token（含行列位置）。

## 3. 语法与语义分析（Parser）

- 已实现递归下降语法分析主流程：
- `PROGRAM -> 声明部分 -> 复合语句 -> .`
- 已实现类型声明：
- `record` 类型定义
- `array[low..high] of T` 类型定义
- 已实现变量声明与作用域信息记录（全局、函数局部、临时变量）。
- 已实现函数声明与调用（值参）。
- 已实现语句：
- 赋值语句
- `if ... then ... [else ...]`
- `while ... do ...`
- 复合语句 `begin ... end`
- 函数调用语句
- 已实现表达式与条件：
- `+ - * /`
- `< > = !=`
- 已实现类型检查（含赋值兼容与函数实参与形参匹配检查）。
- 已实现符号表系统数据构建并输出到结果结构：
- `SYNBL`、`TYPEL`、`RINFL`、`AINFL`、`PFINFL`、`CONSL`、`LENL`、`VALL` 所需信息。

## 4. 数组与结构体（当前状态）

- 已实现数组类型声明与数组变量声明。
- 已实现数组下标语法与类型检查（下标必须是 `integer`）。
- 已实现结构体字段访问语法（`a.b`）与字段合法性检查。
- 已实现“数组元素作为左值/右值”的前端表示（例如 `a[i]` 可参与赋值和表达式）。

当前限制：

- 暂不支持结构体数组字段访问（如 `a[i].f`）。
- 运行时层面对数组/结构体的内存寻址尚未做成真正 VM 语义（目前仍以前端表达和展示为主）。

## 5. 中间代码（四元式）

- 已实现四元式生成，包含：
- 程序边界：`program`、`end`
- 赋值：`:=`
- 算术：`+ - * /`
- 条件与循环控制标记：`if el ie wh do we`
- 关系运算结果生成（用于条件判断）
- 函数调用相关：`param call ret`
- 已实现基本块划分（用于优化与目标代码阶段）。

## 6. 中间代码优化

- 已实现按基本块局部优化：
- 常量折叠
- 常量/复制传播（别名传播）
- 公共子表达式消除（块内）
- 代数化简（如 `x+0`、`x*1`、`x*0`、`x/1`）
- 无用临时赋值删除（基于块内逆序活跃分析）
- 已实现优化前后四元式对照展示。

## 7. 目标代码生成（展示用）

- 已实现从优化后四元式生成目标指令轨迹。
- 已实现映射与生成的核心指令形态：
- `LD R`
- `ST R`
- `ADD/SUB/MUL/DIV R`
- `LT/GT/EQ/NE/LE/GE R`
- `FJ R`
- `JMP _`
- `CALL`
- `RET`
- 已实现目标代码跟踪表（基本块、四元式、目标代码、RDL、SEM）展示。
- 已实现“展示层与运行层双输出”：
- 展示层：`targetTrace`（UI 展示用）
- 运行层：`vmInstructions`（结构化 VM 指令，字段为 `op/a/b/c`）
- 已实现 VM 执行器 `VirtualMachine`，可执行运行层指令并产出变量结果。
- 已在 `CompilerFacade` 接入 VM 执行；若 VM 执行异常，会回退到原四元式解释器。

当前限制：

- VM 目前优先覆盖数值变量、分支循环、函数调用值参主流程；数组/record 的真实地址化执行仍待补全。

## 8. 运行执行（Interpreter）

- 已实现四元式解释执行（基础算术路径）：
- `:= + - * /`
- 已实现最终变量值汇总展示。

当前限制：

- 解释器暂未执行控制流与调用相关四元式（如 `if/while/call/ret`）。
- 解释执行结果目前不等同于“完整目标代码 VM 执行结果”。

## 9. 错误处理与可视化反馈

- 已实现词法错误定位（行、列、非法内容）。
- 已实现语法/语义错误抛出并在 UI 显示。
- 已实现编译失败时保留并展示可用阶段结果（如已扫描出的词法表）。
